# PRD 模板参考

编写 PRD 时参考以下结构，根据实际项目灵活调整。

---

## 模板结构

```markdown
# [产品/功能名称] - 产品需求文档

## 1. 概述

### 1.1 背景
- 为什么需要这个功能？
- 解决什么问题？

### 1.2 目标
- 主要目标是什么？
- 成功的衡量标准？

### 1.3 目标用户
- 谁会使用这个功能？
- 用户的特点和需求？

---

## 2. 功能需求

### 2.1 核心功能

#### 功能 A：[功能名]
- **描述**：功能的详细说明
- **用户故事**：作为 [用户角色]，我希望 [做什么]，以便 [获得什么价值]
- **验收标准**：
  - [ ] 标准1
  - [ ] 标准2

### 2.2 次要功能（可选）
...

---

## 3. 用户流程

### 3.1 主流程

[打开页面] → [输入信息] → [点击提交] → [显示结果]

### 3.2 异常流程
- 网络错误：显示重试按钮
- 验证失败：高亮错误字段

---

## 4. UI 设计说明

### 4.1 页面布局

**布局描述**：
- 顶部：导航栏（Logo + 用户头像）
- 中间：主内容区
- 底部：操作按钮

**ASCII 草图**：
```
+----------------------------------+
|  [Logo]              [用户头像]  |
+----------------------------------+
|                                  |
|     [标题]                       |
|     [输入框1: 用户名]            |
|     [输入框2: 密码]              |
|                                  |
|     [登录按钮]  [注册链接]       |
|                                  |
+----------------------------------+
```

### 4.2 组件规格

| 组件 | 类型 | 说明 |
|------|------|------|
| 用户名输入框 | Input | 必填，最大20字符 |
| 密码输入框 | Password | 必填，最小6字符 |
| 登录按钮 | Button | 主按钮，蓝色 |

### 4.3 状态说明

| 状态 | UI 表现 |
|------|---------|
| 加载中 | 按钮显示 spinner，禁用点击 |
| 空状态 | 显示引导文案 |
| 错误态 | 输入框红色边框 + 错误提示 |

---

## 5. 技术要求（可选）

### 5.1 技术栈
- 前端：React + TypeScript + Tailwind
- 后端：Node.js + Express
- 数据库：PostgreSQL

### 5.2 接口需求

POST /api/login
- 请求：{ username, password }
- 响应：{ token, user }

---

## 6. 开放问题

- [ ] 是否需要"记住我"功能？
- [ ] 密码错误次数限制？
```

## ASCII UI 设计指南

因为不使用 Figma，用 ASCII 草图 + 文字描述来传达 UI 设计：

### 常用 ASCII 符号

```
边框：+ - |
按钮：[按钮文字]
输入框：[____输入提示____]
图标：(icon) 或 <icon>
分隔线：---
箭头：→ ← ↑ ↓
```

### 布局示例

**卡片布局**：
```
+------------------------+
|  (头像)  用户名        |
|  ------------------------|
|  内容文字...            |
|  [点赞] [评论] [分享]   |
+------------------------+
```

**列表布局**：
```
+------------------------+
| [x] 任务1              |
| [ ] 任务2              |
| [ ] 任务3              |
+------------------------+
| [+ 添加任务]           |
+------------------------+
```

**表单布局**：
```
标题：[__________________]
描述：[__________________]
      [__________________]
标签：[下拉选择 ▼]

      [取消]  [确定]
```

### 交互标注

用箭头和文字说明交互：
```
[登录按钮] --点击--> 显示加载动画 --成功--> 跳转首页
                                --失败--> 显示错误提示
```

## 编写建议

1. **写给 AI 看**：PRD 是给 AI 生成代码用的，确保信息清晰具体
2. **具体化**：避免"美观的界面"这类模糊描述，用具体尺寸、颜色、间距
3. **ASCII 优先**：复杂布局用 ASCII 草图，比纯文字更直观
4. **状态完整**：描述加载态、空状态、错误态
5. **可验收**：每个功能有明确的验收标准
