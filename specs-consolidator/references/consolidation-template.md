# Consolidated 文档模板

创建或更新 `specs/updatePRD-consolidated.md` 时参考此结构。实际章节可以按项目领域调整，但版本号、时间线、技术债务和后续规划应保持完整。

```markdown

> **状态**: 已部署 / 待实施 / 部分完成
> **基于**: PRD.md vX.Y

## 目录

1. 版本演进概览
```

## Codex 整合要求

- 先用 `rg --files specs` 找出所有候选 PRD/PLAN/consolidated 文档。
- 逐个读取待整合的 updatePRD 文件，不要只依赖文件名或标题。
- 按功能模块组织内容，避免按版本流水账堆叠。
- 同步更新 `PRD.md` 和 `AGENTS.md` 中的版本号、关键技术细节、踩坑记录。
- 清理旧文档前先确认内容已进入 consolidated 文档，并使用 `git rm` 保留历史。
